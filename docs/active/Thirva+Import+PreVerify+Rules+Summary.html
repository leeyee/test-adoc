Thirva Import PreVerify Rules Summary

13 October 2013

[TOC]

注意：下面所列的顺序就是校验的顺序

Participan Worksheet Pre-Verify

LEVEL1

  1. Import data less than 5000 at a time

LEVEL2

  1. Value is Required(Account, AFN, ALN, AEMAIL, ID, FN, LN)

LEVEL3

  1. Grade Enum Check
  2. Number type check(AccountId, ID)
  3. Name Length check(AFN, ALN, FL, LN)
  4. Date type check(Account_DOB, DOB)
  5. Number of family check(Account_email)

Order Worksheet Pre-Verify

LEVEL1

  1. Import data less than 5000 at a time
  2. Import order line less than 5000 for one order at a time

LEVEL2

  1. Value is Required. (All OrderColumnName attr requried is true)
  2. Name max length check(SeasonName, SessionName, TuitionName)

LEVEL3

  1. Name max length check(RegFN, RegLN)
  2. Amount Type check(Tuition_Price, Order_Total)
  3. All order line with the same order id should have same order total
  4. Only one season in the order
  5. Same forigin order have same reg date
  6. Date type check(Order_Reg_Date)
  7. ==Order Registrant DB check==
  8. ==Tuition DB check==

Season Worksheet Pre-Verify

LEVEL1

  1. Only one Season can be imported at a time
  2. when importing season, there must be a session

LEVEL2

  1. Season name value is required.

LEVEL3

  1. Name length check (Season_name)
  2. Date type check (Reg_open_date)
  3. Season Categroy check
  4. ==Season name DB check==

Session Worksheet Pre-Verify

LEVEL1

  1. Duplidate Check(Session_name)
  2. Import session less than 250 at a time

LEVEL2

  1. Value is required(All SessionColumnName attr requried is true)
  2. Early bird price check
  3. Session Option Required check
  4. Age Restriction check
  5. Grade Restriction check

LEVEL3

  1. Name length check
  2. Description length check
  3. Session capacity check
  4. Wailting list policy length check
  5. Max price check
  6. Session option capacity check
  7. Date type check(start,end,close_reg)
  8. Early bird expired date force check
  9. Overnight check
  10. Overnight column check
  11. Time type check
  12. Start,end logic check
  13. Gender check
  14. Age Range check
  15. Age from to check
  16. Age as of check
  17. Display status check
  18. Session day of ** check
  19. Grad type check
  20. Grade from to check
  21. Grad restriction check
  22. ==Session name DB check==
  23. ==GL code DB check==
  24. ==Location Name DB check==
  25. ==Session type name DB check==
  26. ==Session option type DB check==
  27. ==Session Date Cross check==

Tuition Worksheet Pre-Verify

LEVEL1

  1. Duplicate check(Season_name)
  2. Import tuitions less than 250 at a time

LEVEL2

  1. Value is required(All TuitionColumnName attr requried is true)
  2. Early bird price check

LEVEL3

  1. Name length check
  2. Max price check
  3. Early bird expired date force check
  4. Display status check
  5. Tuition capacity check
  6. Description length check
  7. ==GL code DB check==
  8. ==Tuition Cross check==
